**v1.8.4** — Two fixes. (1) **Structures now spawn mobs.** The old creation-time spawn was a dead end: structures sit 95–350 blocks from the platforms, and Bedrock instantly despawns hostiles >128 blocks from any player, so guards vanished before anyone arrived (chests, being blocks, survived — hence "loot but no enemies"). Removed the creation-time spawn; added `spawnStructureMobs()`, a proximity sweep run from `gameTick` every `STRUCT_MOB_CHECK` (10) game-ticks. For each structure with a player within `STRUCT_MOB_RADIUS` (40, 3D), it refills toward `STRUCT_MOB_CAP` (5) at most once per `STRUCT_MOB_COOLDOWN` (40 game-ticks ≈ 8s), counting existing `monster`-family entities so it tops up rather than piles on. Mobs spawn only on real footing (`findStructSurface`, never over void) and are **non-persistent** — they despawn naturally when the player leaves, so idle structures cost nothing (no nameTag/persistence, per design). `fireproof` still grants fire resistance while engaged. Structure records now persist `h` (height, from `.mcstructure .size.y`) for the surface scan; old records without `h` default to 48. Chest placement/loot at creation is unchanged. (2) **tower_1 was only half a structure** — the build had been exported in two parts and only the 30×35 top shipped. Merged the uploaded 30×25 base under it into one 30×59×30 file (1-layer overlap: base's top face = tower's base, confirmed 39/39 identical at matching x,z), cleaned to FF spec (dropped the structure_block export anchor, 10 barrels incl. 2 with loot, 3 stray dropped items; pruned the now-unused palette entry). The full base also fixes tower_1's old off-center bounding box, so its chest/mob scans now hit solid floor. Old top-only file backed up to the outputs folder.
**v1.9.0** — Three data-driven structure systems, all additive/backward-compatible (existing defs lacking the new fields fall back to prior behavior). (1) **Weighted structure pity.** Selection was flat `pick(avail)`, so the same structure could land several times running. Added an optional `baseWeight` per def (default 10) and a persisted running-weight map (`ff:structWeights`). `pickWeightedStructure()` selects by current weight; `commitStructurePity()` — called only after a placement actually succeeds — zeroes the chosen type and climbs every other available type by its `baseWeight`, so commons recover fast and low-weight builds stay rare. Seeds newly-`minPhase`-unlocked types to the running average (not starved, not instantly favored) and falls back to uniform if all weights hit 0. `baseWeight` doubles as a rarity dial: `casco_marry` set to 3. (2) **Authored mob spawn points.** Optional `def.spawns` = `{x,y,z}` LOCAL offsets from the `.mcstructure` MIN corner (feet position). `spawnStructureMobs()` resolves the origin from a cached `.size` lookup (`getStructSize`), spawns at the authored points with per-point footing validation, and falls back to the existing surface scan when `spawns` is absent or a point is bad. Better encounter design than the scan, which was never broken — the real win is control, not correctness. (3) **Multi-chest support.** Optional `def.chests` = array of `{x,y,z, rarity?, slots?}` LOCAL offsets; each places a chest with an optional per-chest rarity override and an optional slot/loot-roll count (`fillChest` gained a `slotOverride` accepting a scalar or `[min,max]`). No surface scan for authored chests — the offset is owned by the author, so a bad one just buries a chest. Omit `chests` to keep the single auto-placed, surface-scanned chest. (Explicit per-chest item lists were considered and dropped — they'd fork the loot economy off the day-gated tier tables for little gain.) All 10 structures now ship authored `spawns` (and `chests` on the 8 larger/multi-level builds; the two single-room houses keep the auto-scan), generated by parsing each `.mcstructure`'s little-endian NBT for valid standing surfaces (solid footing + 2 air), filtered to real walkable floors (≥4 same-level floor neighbours, interior margin), then spread via farthest-point sampling for mobs and k-means centroids for chests. Every authored point was re-validated against the block data (footing below, air at spot, in-bounds). Counts: spawns 4–12, chests 2–3, scaled to footprint/height. Run `node --check` on the Windows side before building to be safe.
